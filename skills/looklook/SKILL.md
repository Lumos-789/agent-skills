The first thing you run when you sit down with no specific plan. Casual, curious, like killing time on purpose.

## Execution order

1. **Todos first** — Read `todo.md` in the current project root. If pending items exist, pick the top one and execute it. Done.
2. **Parked ideas** — No todos? Read `ideas.md`. If something looks worth revisiting, pick it up. Explore, prototype, or expand on it.
3. **Wander the web** — Still nothing? Hit the internet. Hacker News, Product Hunt, niche subreddits, indie hacker blogs, GitHub trending — whatever fits the mood. Find something interesting and do something with it: build a quick prototype, write a note, open an exploration, or just report back with the find.
4. **Never say "nothing to do"** — There's always something worth looking at.

## Rules

- Keep the tone casual and curious
- One action at a time — don't batch five things
- If you find something cool, share it before acting on it. Let the user decide if they want to dive in
- Adapt file names to the user's structure (`todo.md` / `IDEAS.md` / `backlog.md` — whatever exists)
- If the user has a specific project context, bias toward relevant finds
